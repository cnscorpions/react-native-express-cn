
大部分应用有一些动态数据：用户数据，图片，文档，联系人等等。

管理整个应用的数据有好几种方式，并且最好的选择取决于应用的复杂度。

有疑问的时候，从简单的开始，并且随着应用复杂度上升，使用更高级的替代。

# 管理数据的常见选项

| 选项              | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Component State** | 在整个应用中，把数据存储到组件的`state`中是最简单的方式。每次用户与应用交互，更新组件的状态，或者使用`props`函数更新父级组件的状态。在应用根组件中维护所有状态是有优势的，以便所有状态在一个地方管理，让应用中推断状态变得更容易，如果需要的时候，也方便切换到Redux。
| **Redux**           | [Redux](https://facebook.github.io/react-native/docs/getting-started.html)是中大型状态管理的最常见选择。Redux提供了一个`store`对象，它维护着应用的状态，当状态发生变更时候，它会通知你的React组件。Redux的设计考虑到React，提供了官方的React绑定[React Redux](https://github.com/reactjs/react-redux)。围绕Redux还有一些额外的工具提供了异步事件处理，状态持久化以及更强大的debugging功能。
| **GraphQL**         | [GraphQL](http://graphql.org) `GraphQL`是一个查询语言，用于从支持GraphQL的服务器拉取和更新远程数据。<span style={{ fontWeight: 700, color: 'red'}}>不熟悉GraphQL，所以不好翻译</span> GraphQL is designed to simplify communication between server and client: compared to traditional REST APIs, GraphQL typically requires fewer roundtrips (better performance) and fewer endpoints (enabling faster iteration cycles). To use GraphQL, you'll need to configure your server to handle GraphQL queries, or use a hosted solution like [Graphcool](https://graph.cool). It's common to use Redux along with GraphQL, if you have a significant amount of client-side data in addition to your server-side data. |
| **Realm**           | [Realm](https://realm.io/docs/react-native/latest/) 是一个自定义的数据库，用C语言从头开始写的，兼容iOS和Android双端。Realm适合数据量巨大的app，如1万+条记录。如果app无法将其所有数据存入RAM中，使用数据库可以轻松地将记录分页移入和移出。此外，得益于数据库索，搜索变得非常快，离线使用也不是问题。
| **Firebase**        | [Firebase](https://firebase.google.com/) 是一个可以存储和更新数据的云服务。在没有现成的后端或者API服务情况下，Firebase会是一个好的选择，因为你可以在无基础设施的情况下创建一个移动app。Firebase实时数据同步功能很优秀。该服务还提供了一个Web管理面板，用于查看/修改数据，用户帐户和身份验证，分析，崩溃报告等。

让我们更深入地研究这些选项。
