
客户端数据持久化对于优秀的移动体验来说往往是至关重要的：记住用户的偏好和凭证，并在应用启动时立即显示数据，而不是在远程获取数据时显示一个loading标识。

一般来说，客户端上的数据持久化有两种方式。

- 内置的 "AsyncStorage "API，或封装它的库。
- 一个SQLite/原生数据库的wrapper

# 客户端持久化常见选项

| 选项            | 描述                                                                                                                                                                                                                                                                                    |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **AsyncStorage**  | `AsyncStorage`是一个简单的、低级别的键值存储API。你可以轻松地存储和检索键值对。                                                                                                                                                                          |
| **Redux Persist** | [Redux Persist](https://github.com/rt2zz/redux-persist) 是一个库，用于在store变化时自动将Redux store的状态持久化到 "AsyncStorage"，并在应用启动时用保存的状态为store rehydrating。                                                       |
| **Realm**         | [Realm](https://realm.io/docs/react-native/latest/) 库使用的是自定义数据库，从头开始用C语言编写，在iOS和Android上都能兼容。我们在数据管理下单独介绍了Realm，但如果你使用Realm来管理数据，你就可以任意使用。
