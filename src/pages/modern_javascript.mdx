
在过去，你只能在网页的header中插入`<script>`标签，然后你的JS代码会预期运行。

如今，我们**预处理**JS代码来使用实验性功能以及像JSX这样的语言扩展。

# Babel

Babel 是用来预处理JS的主要工具。

它是高度可配置的编译器，让你可以使用实验性的语言特性和扩展，把它们编译成低版本的JS代码，支持在广泛的平台上运行。

当然，如果一个原生的平台不支持一个ES6特性，像`Promise()`，在很多情况下，Babel可以**polyfill**遗失的API以提供这个功能。

通过引入**source maps**和编译后的JS代码，Babel让你能方便地调试未被编译前的源代码。

JS解释器执行编译后的代码，在debugger中把它映射到源代码中去，使得你能够debug源代码而非，通常非常ugly的编译结果。

# Babel 配置

通过在项目根目录下引入一个`.babelrc`文件，你能够配置Babel。这个文件指定哪些实验性特性能启用，哪些插件能使用（本教程以JSX为例）。

React Nativ为你妥善配置好了Babel，所以你通常不需要创建`.babelrc`文件。然而，如果你想要修改默认的预设置，你可以使用一个npm包，`metro-react-native-babel-preset`，作为基础配置，然后在上面添加额外的插件/配置。
